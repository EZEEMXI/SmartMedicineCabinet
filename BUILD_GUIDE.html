一、环境准备

1.1 需要安装的软件

软件版本要求用途
Qt 6≥ 6.5 (推荐 6.7+)框架本体
Android NDKr26+ 编译 C++ → ARM 机器码
Android SDKAPI Level 34Android 构建工具
JDK17 (必须)Android 签名工具
OpenSSL for Android3.x (Qt 6.5+ 自带)SSL (如需 MQTTS)
Qt 6.5+ 已经内置了 OpenSSL for Android，你不需要额外安装。我们使用 MQTT 1883 端口（非 SSL），不需要 SSL。

1.2 检查 Qt6 是否安装了 Android 组件

打开 Qt Maintenance Tool（在 Qt 安装目录下），确认以下组件已勾选：

必须安装的组件：

  • Qt 6.x.x → Android (arm64-v8a) — ARM64 库
  • Qt 6.x.x → Android (x86_64) — 模拟器用（可选）
  • Qt 6.x.x → Additional Libraries → Qt SQL
  • Qt 6.x.x → Additional Libraries → Qt Network

如果没有勾选，用 Maintenance Tool 添加，需要下载约 2-3 GB。

1.3 配置 Qt Creator 的 Android 设置

打开 Qt Creator → 编辑 → Preferences → 设备 → Android：

  1. JDK location：指向 JDK 17 的安装路径
    如 C:\Program Files\Java\jdk-17
  2. Android SDK location：可以用 Qt Creator 自带的 SDK Manager 下载，或者手动指定
    如 C:\Users\你的用户名\AppData\Local\Android\Sdk
  3. Android NDK：Qt Creator 会自动检测。如果没检测到，手动指定
    如 C:\Users\你的用户名\AppData\Local\Android\Sdk\ndk\26.1.10909125
如果你还没有 Android SDK/NDK，可以在 Qt Creator 的 SDK Manager 里一键下载。打开：Tools → SDK Manager。

二、构建 APK

2.1 打开项目

1 启动 Qt Creator

2 File → Open File or Project，选择 CMakeLists.txt

3 选择构建套件：勾选 Android Qt 6.x.x Clang arm64-v8a

2.2 配置 CMake

Qt Creator 会自动运行 CMake 配置。如果提示缺少模块，检查是否安装了对应的 Qt 组件。

2.3 编译

4 点击左下角的 🔨 构建 按钮（或 Ctrl+B）

编译成功后会在输出目录生成 libSmartMedicineCabinet_arm64-v8a.so。

2.4 生成 APK

5 Build → Build APK（或点击 🚀 运行按钮右侧的下拉菜单选择 "Deploy"）

Qt Creator 会调用 androiddeployqt 工具自动生成 APK。APK 位置：

build/SmartMedicineCabinet-Android_Qt6_...-Release/android-build/SmartMedicineCabinet.apk
如果只用 Build（不 Deploy），APK 在 build 目录下。如果点击运行（Run），Qt Creator 会自动安装到 USB 连接的手机上。

三、USB 部署到手机

3.1 手机端设置

  1. 打开 设置 → 关于手机
  2. 连续点击 "版本号" 7 次 → 开启 开发者模式
  3. 设置 → 开发者选项：开启 USB 调试
  4. 用 USB 数据线连接电脑
  5. 手机上弹出的 "允许 USB 调试？" 对话框 → 点击 始终允许

3.2 安装

# 方法1：Qt Creator 里直接点运行
# 在 Qt Creator 中选择 Android 套件 → 点击 ▶ 运行

# 方法2：用 adb 命令行安装
adb install build/.../android-build/SmartMedicineCabinet.apk

四、首次使用

安装 APK 到手机后，首次打开：

  1. 在登录页输入 OneNet 三要素（产品ID / 设备名 / 密钥）
  2. 输入昵称，点击 "连接并进入"
  3. MQTT 连接成功后自动跳转主页
  4. 底部 4 个 Tab：首页(药品) | 健康 | 购物车 | 我的

五、可能遇到的问题

问题1：CMake 找不到 Qt6

CMake Error: Could not find a package configuration file provided by "Qt6"

解决：在 Qt Creator 中选择正确的 Kit（Android Qt6），确保 Qt6 安装完整。

问题2：编译时报 "moc" 相关错误

AutoMoc error

解决：检查 CMakeLists.txt 中 CMAKE_AUTOMOC ON 是否开启。确保所有 Q_OBJECT 类的头文件都已声明。

问题3：APK 安装后闪退

解决：用 adb logcat 查看日志：

adb logcat | grep -i "Qt\|SmartMedicine"

常见原因：缺少 Android 权限、QML 文件路径错误、SQL 驱动未加载。

问题4：MQTT 连不上

解决：

  • 确认手机 WiFi 已连接互联网
  • 确认 OneNet 设备已激活（在线状态）
  • 确认三要素正确（产品ID、设备名、密钥）
  • 在 AndroidManifest.xml 中已经声明 INTERNET 权限 ✓
  • 如果使用 1883 端口（非 SSL），确认 usesCleartextTraffic="true" ✓

问题5：Qt Creator 检测不到手机

解决：

# 在命令行执行
adb devices

# 应显示类似：
# List of devices attached
# 1234567890ABCDEF    device

# 如果没有显示，尝试：
adb kill-server
adb start-server
